# Documentation Conventions (internal — not shipped)

## Tool Call Notation
- Show as: `tool_name(param: "value", param2: "value")`
- ALWAYS include `document` parameter in examples
- Use consistent placeholder: `document: "MyDoc"`

## Comment Syntax in Code Blocks
- Use `#` for all comments (code blocks are pseudo-syntax, not runnable Tinderbox)

## Path Placeholders
- Use `/path/to/note` for generic note references
- Use real paths where relevant: `/Prototypes`, `/Templates`, `/Hints`

## Terminology (exact phrasing)
- Tools: the `evaluate` tool, the `do` tool, the `set_value` tool, the `create_note` tool, the `get_notes` tool, the `create_link` tool
- Expression: "expression" (read-only, returns value, used with `evaluate` tool)
- Action: "action code" (mutates state, used with `do` tool)
- Query: "query" (only for `$AgentQuery` or `get_notes` query parameter)
- Note reference: "note path" when referring to the `/path/to/note` string

## Boolean Return Values (authoritative)
- Comparisons: `"true"` for true, `""` (empty string) for false. NEVER `"false"`.
- `.contains()` on strings: returns 1-based character position (integer), or `0` if not found
- `.contains()` on sets/lists: returns 1-based item position, or `""` (empty string) if not found
- In boolean context, both `0` and `""` evaluate as false

## Section Structure (every reference file)
1. H1 title
2. Brief intro paragraph (1-2 sentences explaining what the file covers)
3. Note: "Examples include the `document` parameter. Replace `"MyDoc"` with your document name."
4. Content sections (H2/H3)
5. `---` separator
6. `## Cross-References` section with links to related files

## Date Format Codes
- Zero-padded day: `D0` (not `D` — which is also zero-padded but the `0` clarifies)
- Zero-padded month: `M0`
- In examples always use `D0` for clarity
